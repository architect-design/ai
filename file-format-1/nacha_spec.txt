NACHA ACH FILE FORMAT SPECIFICATION
Version 2024 – NACHA Operating Rules
=====================================

OVERVIEW
--------
The ACH file is a fixed-width flat file. Every record is exactly 94 characters
wide followed by a newline. Records are grouped into blocks of 10 lines; if
the total line count is not a multiple of 10, pad with "9" characters to fill.

RECORD TYPE CODES
-----------------
1  File Header Record       (one per file, first record)
5  Batch Header Record      (one per batch)
6  Entry Detail Record      (one per transaction)
7  Addenda Record           (optional, follows Entry Detail)
8  Batch Control Record     (one per batch, closes batch)
9  File Control Record      (one per file, last data record)

=================================================================
VALIDATION RULES
=================================================================
1. Every record must be exactly 94 characters (excluding line terminator).
2. File total lines must be a multiple of 10 (pad with '9' * 94 if needed).
3. Entry Hash in Batch Control = sum of first 8 digits of all RDFI routing numbers in that batch,
   take only the rightmost 10 digits.
4. Entry Hash in File Control = same, summed across all batches.
5. Debit/Credit totals in File Control = sum across all batches.
6. Batch numbers must be sequential starting at 0000001.
7. Trace number = ODFI routing (8 digits) + 7-digit sequence number within batch.
8. ABA routing number 9th digit must satisfy: (3*d1 + 7*d2 + d3 + 3*d4 + 7*d5 + d6 + 3*d7 + 7*d8 + d9) mod 10 = 0
